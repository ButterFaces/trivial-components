Beautiful Components

Beauty is important. Therefore we design our components to look delicious. The design integrates well with Twitter Bootstrap.

No Feature Compromises

Trivial components make no compromises concerning their functionality. However, we do make compromises in browser support.

Trivial to use

All components are provided as jQuery plugins with sensible defaults. The showcases give you the possibility to try them out.

Trivial Components should be considered alpha state software. We are working hard for our first stable release.

Showcases

ComboBox

Zero Configuration

$('#zeroConfigComboBox').TrivialCombobox({
    entries: myEntriesArray
});

Custom filtering

Filtering is done by custom code, e.g. on the server side.

$('#customFilteringComboBox').TrivialComboBox({
    queryFunction: function (queryString, resultCallback) {
        // custom processing, maybe on the server side
        ...
        resultCallback(entriesFromServer);
    }
});

Templates

There are 3 of-the-shelf templates. However, you can write any template you like yourself.

Icon (25x30) with two lines of text
$('#icon2LinesTemplateComboBox').TrivialComboBox({
    template: $.fn.TrivialComboBox.icon2LinesTemplate
    ...
});
Icon (24x24) one line of text
$('#iconSingleLineTemplateComboBox').TrivialComboBox({
    template: $.fn.TrivialComboBox.iconSingleLineTemplate
    ...
});
Single line of text
$('#singleLineTemplateComboBox').TrivialComboBox({
    template: $.fn.TrivialComboBox.singleLineTemplate
    ...
});

Other options

autoComplete: false
$('#noAutoCompleteComboBox').TrivialComboBox({
    autoComplete: false
    ...
});
showTrigger: false
$('#noTriggerComboBox').TrivialComboBox({
    showTrigger: false
    ...
});

Configuration

Option Description Default
valueProperty Name of the property which should be submitted on form submit. If null, then the content of the text input will serve als value. 'displayValue'
allowFreeText Allow free text entry. If true, the user is not forced to select one of the suggested entries but can type whatever he wants into the field. If the user leaves the field without selecting any of the suggested values, then his input remains visible. If false, the user input will be deleted if he does not select a suggested entry. false
inputTextProperty Name of the property which will be filled into the input field when the user selects an entry. This property is also used for auto-completion. 'displayValue'
template Mustache template to be used for displaying the combobox dropdown entries. defaultTemplate
selectedEntryTemplate Mustacheustache template to be used for displaying the currently selected entry. template
spinnerTemplate Html code for the "loading..." indicator. defaultSpinnerTemplate
noEntriesTemplate Html code for indicating that there is no entry in the dropdown. defaultNoEntriesTemplate
entries The dropdown entries to be displayed initially. []
selectedEntry The initially selected entry. undefined
emptyEntry Fake entry that should be displayed when no entry is selected. {}
queryFunction JavaScript function for retrieving entries depending on the user's input. Takes two parameters: queryString (user input) and resultCallback (should be called with the list retrieved entries). Simple client-side implementation that filters the initially configured array of entries.
autoComplete If true, the component user get auto-completion inside the text field. true
autoCompleteDelay Delay for auto-completion. (This feature may be removed, as there is currently no reason for a delay in auto-completion.) 0
showTrigger The trigger is the button-like handle on the side of the combobox. If false: no trigger is shown. true
